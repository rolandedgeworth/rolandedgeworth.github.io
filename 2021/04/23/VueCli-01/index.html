在标签上使用

在标签上使用变量需注意两点

  • :style="mainStyle" style 前的冒号（:），有冒号时，mainStyle 将被按照 JS 解读，没有冒号时，mainStyle 将被作为一个字符串传入
  • mainStyle 而不是 this.mainStyle ，在 <script> 标签内，this 多指向当前组件的 Vue 实例，而在一个标签上，this 读取到的将会是 null

在 vue 指令中使用

在 vue 指令 中（v-xxx）使用时，即使没有冒号，其取值也将被按照 JS 语句来解析

常用指令 v-xxx

vue 指令，即在 html 节点内使用，形如 v-xxx 的特殊属性

v-bind

在前文中，有这样一个示例

其中的 :style 是一种简写，其全写为 v-bind:style

v-bind 指令用于绑定 html 属性与数据，将 script 中的数据作为 html 属性的取值，其中的 html 属性包括原生属性与自定义属性，而数据则可以是任意类型数据，包括 data、computed、methods 等

v-model

v-model 可以实现 html元素 与数据的双向绑定

当使用 v-model 将变量 value 与 input 元素绑定后，无论在 DOM （页面上直接编辑）上还是在 JS 中修改，页面上 input 输入的值与 变量 value 的值都会实时性地保持一致

实际开发时，我们一般将其用于表单输入中，包括输入、单选、多选、下拉选择等

v-on

v-on 用于绑定事件，将一段处理逻辑与事件进行绑定

v-on:click="btnClickHandler" 将其所在的元素上的 click 事件与方法 btnClickHandler 进行了绑定，即这一 button 元素的 click 事件每次触发时，会对应执行一次 btnClickHandler 方法

与 v-bind 类似， v-on 也有简写方式，比如对于 v-on:click="btnClickHandler"，我们一般将其简写为 @click="btnClickHandler"，效果完全一致

v-for

v-for ，请将其理解为 for 循环，循环遍历性地把一个数组对应为一组元素

在 v-for="item in arr" 中， item 即为对数组进行循环时得到的每一个元素，第一次循环时，item 为 1，第二次为2，以此类推

在实际开发时，为避免渲染发生错误，一般（必须）要在 v-for 所在的节点上指定一个 key 属性，其取值为一个唯一值，不能重复
